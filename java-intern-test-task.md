# Тестовое задание (Java)

## Сервис учёта рабочего времени сотрудников — «Task Time Tracker API»

## Контекст

В компании необходимо вести учёт времени, которое сотрудники тратят на задачи. Вам необходимо разработать backend REST-сервис на Java и Spring Boot, который позволит создавать задачи и фиксировать временные отрезки, затраченные на них.

## Функциональные требования

### Предлагаемые сущности

- **Task** — ID, Название, Описание, Статус: `NEW` / `IN_PROGRESS` / `DONE`
- **TimeRecord** — ID, ID сотрудника, ID задачи, Время начала, Время окончания, Описание работ

### Необходимые эндпойнты REST-сервиса

- Создание задачи
- Получение информации о задаче по ID
- Изменение статуса задачи
- Создать запись о затраченном времени сотрудника на задачу
- Получить информацию о затраченном времени сотрудника на задачи за определённый период времени

## Базовый стек технологий

`Java 17+` · `Spring Boot 3.x` · `MyBatis` · `Maven` · `JUnit 5` · `Mockito`

## База данных

Можно использовать встроенную БД **H2** (для упрощения) или **PostgreSQL** (Docker).

## README

Необходимо сделать краткое описание сервиса в README (какой функционал реализован, как проверить работоспособность) и чёткую инструкцию, как собрать и запустить приложение (а также БД, если используется вариант с PostgreSQL). Приложите к проекту набор тестовых запросов, демонстрирующих работу всех реализованных REST-эндпойнтов (например, коллекцию Postman).

Обязательно покрытие **Unit-тестами**.

## Дополнительные критерии

Выполнение следующих пунктов учитывается как дополнительный критерий отбора:

- Декларативная документация API через **SpringDoc OpenAPI (Swagger)**
- Входные DTO должны валидироваться (**Bean Validation**)
- Обработка исключений, в том числе через `@RestControllerAdvice`
- Реализация интеграционного теста взаимодействия с БД — DAO-слоя приложения (**TestContainers**)
- Применение **Bearer Authentication (JWT)**
- Использование ИИ в качестве помощника при выполнении тестового задания приветствуется; желательно сопроводить итоговый проект артефактами взаимодействия с чат-ботом / ИИ-агентом (например: скриншоты основных шагов, промты)
